DoctoGoogle — Plateforme Cloud de gestion automatisée (Google Workspace)

DoctoGoogle est un projet cloud-native développé autour de Google Workspace et Google Apps Script. L’objectif est de créer une plateforme automatisée de gestion de cours et de formations, sans infrastructure serveur dédiée, entièrement basée sur des services SaaS.

Contexte du projet

Le projet a été réalisé dans un cadre académique et professionnel pour une structure utilisant déjà Google Workspace comme système d’information.

La contrainte principale était de concevoir une solution 100 % cloud, intégrée, sécurisée et maintenable, sans recours à un backend traditionnel (pas de serveur, pas de base SQL dédiée).

Comparaison et positionnement technique

DoctoGoogle peut être comparé à :

  • Zapier / Make — pour l’orchestration de services
  • Microsoft Power Automate — pour l’automatisation métier
  • Mini‑ERP SaaS — agenda, base de données, workflow, notifications

La différence majeure est que l’ensemble est développé sur mesure via Apps Script (JavaScript serverless), directement dans l’écosystème Google.

Architecture Cloud & modules

  • Google Forms — interface utilisateur (inscriptions)
  • Google Sheets — base de données centrale (conteneur Apps Script)
  • Google Calendar — planification et synchronisation des événements
  • Gmail — notifications automatiques
  • Apps Script — moteur logique & orchestration

Mes missions principales

Responsabilité technique & coordination

J’occupais un rôle central dans le projet, avec une responsabilité de pilotage technique et de coordination entre modules.

  • Définition de l’architecture globale
  • Découpage fonctionnel en modules Apps Script
  • Centralisation du code dans un conteneur unique (Sheets)
  • Synchronisation inter‑services Google

Développement — Module Agenda

  • Création automatique d’événements Google Calendar depuis la base de données
  • Synchronisation bidirectionnelle Agenda ↔ Sheets
  • Gestion des formats de dates / horaires
  • Ajout de métadonnées (enseignant, matière, email)

Orchestration inter‑modules

Le cœur du projet repose sur la coordination entre modules indépendants :

  • Formulaire → Base de données
  • Base de données → Agenda
  • Agenda → Mise à jour dynamique du formulaire
  • Inscription → Envoi automatique d’e‑mail

Aspects développement

  • JavaScript serverless (Apps Script)
  • Déclencheurs événementiels (onOpen, time‑based, actions utilisateur)
  • Manipulation d’API Google (Calendar, Gmail, Forms, Sheets)
  • Gestion des données structurées sans base SQL
  • Automatisation complète de workflows métier

Travail en équipe (anonymisé)

Le projet a été réalisé en équipe, avec une répartition modulaire :

  • Développeur A — gestion dynamique des formulaires
  • Développeur B — automatisation des notifications e‑mail
  • Moi — architecture, agenda, intégration et coordination

Résultats & apprentissages

  • Conception d’une architecture cloud sans serveur
  • Approche modulaire et découplée
  • Gestion des dépendances inter‑services
  • Vision produit et workflow métier
  • Expérience concrète en automatisation SaaS

Limites & perspectives

Les limites sont principalement liées aux quotas Apps Script et aux contraintes de l’écosystème Google.

  • Ajout d’une interface front dédiée
  • Gestion avancée des droits utilisateurs
  • Journalisation / audit des actions
  • Scalabilité via comptes Workspace étendus
