Projet Cloud — Azure & AWS (IaaS / PaaS)

Ce projet Cloud avait pour objectif principal la découverte, la prise en main et la compréhension concrète des environnements Microsoft Azure et Amazon Web Services (AWS). Il s’inscrit dans une démarche de montée en compétences progressive, couplée à une production de documentation technique destinée à être réutilisée par les futurs étudiants du cursus.

Objectifs du projet

  • Découvrir les concepts fondamentaux du Cloud (IaaS, PaaS, services managés)
  • Manipuler concrètement les services Azure et AWS
  • Comprendre les mécanismes réseau, sécurité et interconnexions
  • Produire une documentation technique claire et exploitable
  • Créer un support de référence pour les promotions suivantes

Périmètre technique exploré

Infrastructure as a Service (IaaS)

  • Création et gestion de machines virtuelles Linux & Windows
  • Réseaux virtuels, sous-réseaux, adressage IP
  • Groupes de sécurité réseau (NSG / Security Groups)
  • Accès distants : SSH, RDP

Platform as a Service (PaaS)

  • App Services / hébergement web managé
  • Déploiement d’applications simples
  • Services de bases de données managées

Sécurité & réseau

  • Filtrage réseau entrant / sortant
  • Segmentation réseau
  • Interconnexion entre réseaux virtuels
  • VPN, passerelles et NAT
  • Pare-feu Cloud (Azure Firewall)

Mes missions principales

Prise en main Cloud

  • Exploration des portails Azure et AWS
  • Utilisation conjointe du portail graphique et des CLI
  • Compréhension des coûts, des quotas et des bonnes pratiques

Rédaction de documentation technique

Une part essentielle du projet consistait à formaliser les manipulations sous forme de documents techniques détaillés :

  • Procédures pas à pas
  • Captures d’écran commentées
  • Commandes CLI expliquées
  • Schémas d’architecture

Ces documents ont vocation à servir de support pédagogique pour les étudiants qui choisiront ce projet à l’avenir.

Travail en collaboration

Le projet a été mené en binôme, dans une logique de collaboration technique et de partage des connaissances.

  • Moi — coordination technique, Cloud Azure, rédaction principale
  • Collaborateur (anonymisé) — participation aux expérimentations et validations

Cette collaboration a permis de croiser les approches, de confronter les solutions et d’améliorer la qualité des livrables produits.

Apprentissages & apports

  • Compréhension concrète des architectures Cloud
  • Vision claire des différences IaaS / PaaS
  • Importance de la sécurité dès la conception
  • Capacité à documenter un environnement technique complexe
  • Transmission des connaissances à d’autres profils techniques

Limites & perspectives

Le projet était volontairement orienté découverte et compréhension. Les perspectives naturelles seraient :

  • Automatisation avancée (IaC : Terraform, Bicep)
  • Approche DevSecOps
  • Surveillance et journalisation Cloud
  • Durcissement sécurité avancé
